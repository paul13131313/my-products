## ✅ 必ずやること

### データ
- [ ] 問題データは公式サイトからPDFで取得（無料）
- [ ] 正解番号はPDFから正確に取得し、CCに渡す
- [ ] 解説はCCにオリジナル生成させる（著作権回避）
- [ ] 5年分（約250問）以上入れてからリリース
- [ ] 問題投入後、**正答率の計算ロジックを必ず動作確認**（今回バグあり）

### 表記
- [ ] サブタイトルに**「令和○年度対応」**を必ず入れる（最新感・信頼感）
- [ ] **問題数を数字で明示**（例：250問収録）
- [ ] 「過去問」という文言はTOP画面に入れない（審査リスク）
- [ ] App Store説明文：「出題傾向を徹底分析した重要問題○問収録」

### 表示・UI
- [ ] カテゴリ名が**ローマ字になっていないか**確認（今回発生）
- [ ] 問題文・選択肢が長い場合に**折り返し表示されるか**確認
- [ ] 全8分野（または各カテゴリ）に問題が割り当てられているか確認
- [ ] 模擬試験モードのタイマーが正常動作するか確認

## 🔄 横展開手順（テンプレート流用）

1. `questions.json` を新資格のデータに差し替え
2. `categories.json` のカテゴリ名を更新
3. `app.json` のアプリ名・Bundle IDを変更
   - Bundle ID例：`com.paul19820113.{資格名-英字}`
4. ホーム画面のタイトル・サブタイトル変更
   - サブタイトル：「令和○年度対応｜○問収録」
5. App Store用スクリーンショット撮り直し
6. ビルド&提出

## 📋 CC指示文テンプレート（横展開用）

```
以下のPDFをドラッグ&ドロップしています：
- past_question_20XX.pdf × 5年分
- past_answer_20XX.pdf × 5年分

## やること

### 1. questions.jsonを作成
既存フォーマットに合わせて全問変換。
- id："{年度}-{3桁連番}"
- year・category・question・choices・answer・explanation
- answerはpast_answerのPDFから正確に取得（0〜3のインデックス）
- explanationは200字程度のオリジナル解説文を生成
- categoryは日本語で記載すること（ローマ字禁止）

### 2. categories.jsonを{資格名}の分野に更新

### 3. app.jsonのアプリ名・Bundle IDを変更
- name: "{資格名} 完全攻略"
- bundleIdentifier: "com.paul19820113.{英字}"

### 4. HomeScreenのサブタイトルを変更
「令和7年度対応｜{問題数}問収録」

### 5. 正答率の計算ロジックを確認
results画面で正答率が正しく表示されることをシミュレートして確認すること。

### 6. ビルド&提出
eas build --platform ios --profile production --non-interactive && eas submit --platform ios --latest
```
